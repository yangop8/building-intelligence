P01ResNet:让网络能真的"深"起来

对应主书:第 3 章 4.4.1 CNN · 第 4 章 5.2.4(残差连接思想)

Deep Residual Learning for Image Recognition
作者Kaiming He, Xiangyu Zhang, Shaoqing Ren, Jian Sun
机构Microsoft Research Asia
发表CVPR 2016(最佳论文奖)
被引250,000+(CS 领域历史第一)
链接arXiv:1512.03385

问题:深度的悖论

2014 年 VGG-19 证明 19 层 CNN 比 8 层 AlexNet 强很多。直觉上,继续加深应该继续涨点。但实验发现:20 层网络比 56 层网络训练集 loss 更低——不是过拟合(过拟合是训练好测试差),是训练都训不下去。梯度消失/爆炸只是表面原因,更深层的原因是:多层非线性叠加后,优化器难以找到好参数。

直观反驳:理论上深网络不应该比浅网络差——只要让多余的层学成恒等映射 $f(x)=x$ 就行。但实际 SGD 很难训出"恒等映射"的层(需要精确地学到特定权重)。这就是何恺明团队抓住的点。

核心方法:残差块

不学 $H(x)$,学 $F(x) = H(x) - x$,输出 $H(x) = F(x) + x$。

flowchart TB X[x] --> C1[Conv 3x3 + BN] C1 --> R1[ReLU] R1 --> C2[Conv 3x3 + BN] X -. identity .-> ADD((+)) C2 --> ADD ADD --> R2[ReLU] R2 --> OUT[output]

如果"什么都不学"是最优的,让 $F \to 0$ 比让整个复合函数 $H \to$ 恒等容易得多。这是最朴素但最关键的 reframing。

关键公式与初始化

一个残差块:

$\mathbf{y} = \mathcal{F}(\mathbf{x}, \{W_i\}) + \mathbf{x}$

反向传播时,损失 $L$ 对输入 $\mathbf{x}$ 的梯度:

$\frac{\partial L}{\partial \mathbf{x}} = \frac{\partial L}{\partial \mathbf{y}} \cdot \left(1 + \frac{\partial \mathcal{F}}{\partial \mathbf{x}}\right)$

那个 "+1" 是关键——它保证梯度不会衰减到 0,即便 $\partial \mathcal{F} / \partial \mathbf{x}$ 很小。这直接缓解了梯度消失。

另一个重要细节是He 初始化(同作者 2015 年另一篇)。ReLU 下权重按 $\mathcal{N}(0, 2/n_{in})$ 初始化,让前向激活方差保持稳定。没有这个初始化,残差网络也训不了。

PyTorch 实现

import torch.nn as nn

class BasicBlock(nn.Module):
    def __init__(self, in_ch, out_ch, stride=1):
        super().__init__()
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, 1, 1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_ch)
        # 下采样或通道数变化时, 需要 1x1 conv 调整维度
        self.shortcut = nn.Sequential()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride, bias=False),
                nn.BatchNorm2d(out_ch),
            )

    def forward(self, x):
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = out + self.shortcut(x)   # 残差连接
        return torch.relu(out)

Bottleneck 变种

深的 ResNet(50、101、152 层)用 "bottleneck" 块降计算:1×1 conv 降维 → 3×3 conv → 1×1 conv 升维。3 层替代 2 层,但总参数和计算量更少。ResNet-50 用这个,之后所有 CNN 几乎都跟进。

实验结果

  • - ImageNet:34 层 ResNet top-5 错误率 7.5%,VGG-19 是 8.4%,且快得多
  • - 152 层 ResNet 达到 5.7%——第一个超越人类级别 5.1% 的模型
  • - CIFAR-10 上训练了 1202 层网络(就是为了证明"真能深"),虽然过拟合但训练 loss 仍在下降——证明残差思想不受深度限制

影响

ResNet 的残差思想已经不只是 CNN 的特性——它成为所有深度网络的基础组件:

  1. Transformer:每个 attention 和 FFN 模块外都包残差(主书 5.2.1)。没有残差,BERT/GPT 深度训练不可能。
  2. DenseNet、U-Net:残差思想的变种——密集连接、跳跃连接
  3. ControlNet、LoRA:都隐含"旁路 + 残差"的架构哲学
  4. Neural ODE:把残差网络看作 ODE 的离散形式 $\mathbf{y} = \mathbf{x} + \mathcal{F}(\mathbf{x}, t)$

批判视角

2020 年后有研究质疑:ResNet 的成功可能主要来自"集成效应"——残差路径让网络行为像 $2^N$ 个不同深度子网络的集成,而不是真正的"学习残差"。但无论解释是什么,这个架构决策的价值毋庸置疑。

为什么必读:不读 ResNet,你不会真正理解为什么 Transformer 的每个 sublayer 都包一层残差——这不是装饰,是深度网络能训起来的前提。何恺明凭这一篇论文基本定义了"工程上能训的深度网络"的标准样式。

P02LSTM:序列建模的经典守门人

对应主书:第 3 章 4.4.2 RNN/LSTM/GRU

Long Short-Term Memory
作者Sepp Hochreiter, Jürgen Schmidhuber
机构IDSIA / TU Munich
发表Neural Computation 1997
被引90,000+
链接doi:10.1162/neco.1997.9.8.1735

问题:普通 RNN 的梯度消失

Hochreiter 1991 年的硕士论文已经定量证明:标准 RNN 的梯度通过时间反传时,每步乘一个权重 $W$。如果 $W$ 的谱半径 < 1,梯度指数衰减;> 1 则爆炸。两种情况下 RNN 都无法学到"10 步之前"的依赖。6 年后他和 Schmidhuber 提出 LSTM 作为解法。

核心方法:三门控 + 细胞状态

引入一条"高速通道"——细胞状态 $\mathbf{c}_t$,通过三个门控制信息流:

$\mathbf{f}_t = \sigma(W_f \cdot [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f) \quad \text{(遗忘门)}$
$\mathbf{i}_t = \sigma(W_i \cdot [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i) \quad \text{(输入门)}$
$\tilde{\mathbf{c}}_t = \tanh(W_c \cdot [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c) \quad \text{(候选)}$
$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t$
$\mathbf{o}_t = \sigma(W_o \cdot [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o) \quad \text{(输出门)}$
$\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t)$

梯度通道的关键:$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \ldots$ 是加法(不是矩阵乘)。如果遗忘门 $\mathbf{f}_t \approx 1$,$\mathbf{c}_t$ 保留所有历史,梯度几乎无衰减地穿过时间。这是 LSTM 能学长依赖的数学原因。

代码

class LSTMCell(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        # 4 个门合成一个大矩阵, 提高计算效率
        self.W = nn.Linear(input_size + hidden_size, 4 * hidden_size)
        self.hidden_size = hidden_size

    def forward(self, x, state):
        h_prev, c_prev = state
        gates = self.W(torch.cat([h_prev, x], dim=-1))
        i, f, g, o = gates.chunk(4, dim=-1)
        i, f, g, o = torch.sigmoid(i), torch.sigmoid(f), torch.tanh(g), torch.sigmoid(o)
        c = f * c_prev + i * g
        h = o * torch.tanh(c)
        return h, (h, c)

影响与让位

LSTM 统治了 1997-2017 这 20 年的序列建模:语音识别(Graves 的 deep LSTM,2013)、机器翻译(Google NMT 2016)、图像描述(Show-and-Tell 2014)全部基于 LSTM。"attention"机制最早(Bahdanau 2014)就是为 encoder-decoder LSTM 设计的——只是后来人们发现 attention 自己就够,不需要 LSTM 做底。

2017 年 Transformer 让 LSTM 从主流退居到"小模型场景"——但在嵌入式时序、某些专用任务里,LSTM 仍是首选。2023-2024 Mamba 等 SSM 模型的复兴,某种意义上也是 LSTM 精神的现代重生。

有趣的历史

Hochreiter 和 Schmidhuber 的 1997 原论文实际有一些现在看来"次优"的细节——比如没有遗忘门(是 Gers 1999 年补的)。我们今天说的 LSTM 是"Gers-LSTM"而非原版。这对想读原论文的人是个提醒:主流工具往往是累积演化的,不是单篇论文定型的。

为什么必读:理解"门控 + 加性记忆通道"是现代所有"选择性记忆"机制(Mamba 的 selective state、Transformer 的 attention gating)的思想源头。

P03BatchNorm:让深网络训练变简单

对应主书:第 3 章 4.3.5 归一化

Batch Normalization: Accelerating Deep Network Training by Reducing Internal Covariate Shift
作者Sergey Ioffe, Christian Szegedy
机构Google
发表ICML 2015
被引60,000+
链接arXiv:1502.03167

问题:为什么深网络训练对学习率极其敏感

Ioffe 观察到:每层的输入分布随着前面层参数更新而剧烈变化。这叫"internal covariate shift"。结果就是下一层必须不断"追赶"分布变化,训练不稳定;而且不同层最优学习率差几个数量级,全局 lr 调起来痛苦。

核心方法

对 mini-batch 内每个特征通道,做标准化:

$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}, \quad y_i = \gamma \hat{x}_i + \beta$

$\mu_B, \sigma_B^2$ 是 batch 统计,$\gamma, \beta$ 是可学习缩放和平移——允许网络"恢复"到标准化前的分布,只是不再需要前面层的参数被约束。

训练 vs 推理:训练用 batch 统计;推理用"运行均值/方差"(训练时指数平均维护)。这个区别导致了 BN 的 subtle bug 温床——训练集好、推理集差的一个常见原因。

BN 的"效果解释"争议

原论文归因于"减少内部协变量漂移"。但 2018 年 MIT 的 Santurkar 等人的论文"How Does Batch Normalization Help Optimization?"实验证明:covariate shift 和 BN 的效果关系很弱。真正原因更可能是 BN 让损失曲面更平滑,大的学习率也不会让梯度爆炸。

这是深度学习研究一个典型现象:一个方法被证明有效,但后来才发现它有效的原因跟作者说的不一样。

影响

BN 是 2015 年之后 CNN 的标配。让 100+ 层网络训练成为可能,和 ResNet 一起支撑了 CV 的深度时代。

但 BN 对 RNN、Transformer 不好使——batch 维度的统计量对序列任务意义不大。Transformer 全用 LayerNorm,后来又演化出 RMSNorm。所以在主书 5.2.4 里你看到 "Pre-Norm + RMSNorm" 是 LLM 标配。BN 的故事基本结束在 CV 领域。

为什么必读:理解"归一化不是数据预处理,是深度网络的结构组件"——这个观念转变是 BatchNorm 的真正贡献。读这篇论文能让你对主书里 LayerNorm、RMSNorm、GroupNorm 的意义有根子上的理解。

P04Adam:深度学习默认优化器

对应主书:第 1 章 2.8.4 · 第 3 章 4.5.1

Adam: A Method for Stochastic Optimization
作者Diederik P. Kingma, Jimmy Ba
机构University of Amsterdam, University of Toronto
发表ICLR 2015
被引200,000+
链接arXiv:1412.6980

问题:SGD 的学习率调节难

纯 SGD 需要精细的学习率调度——每个参数、每个阶段用不同 lr。AdaGrad(Duchi 2011)按参数历史梯度的累积平方自适应调节——学过的参数 lr 自动小,新参数 lr 大。问题:累积只增不减,训练后期所有参数 lr 都趋近 0,训练停止。RMSProp(Hinton 2012 课程讲义)用指数平均替代累积,解决"永久衰减"。

Adam = RMSProp + momentum,再加个偏差修正。

核心算法

维护两个状态:一阶矩估计(动量)$\mathbf{m}_t$ 和二阶矩估计(梯度平方的指数平均)$\mathbf{v}_t$。

$\mathbf{m}_t = \beta_1 \mathbf{m}_{t-1} + (1 - \beta_1) \mathbf{g}_t$
$\mathbf{v}_t = \beta_2 \mathbf{v}_{t-1} + (1 - \beta_2) \mathbf{g}_t^2$

偏差修正——初期 $\mathbf{m}_0 = \mathbf{v}_0 = 0$,指数平均会偏低,除以 $(1 - \beta^t)$ 补偿:

$\hat{\mathbf{m}}_t = \frac{\mathbf{m}_t}{1 - \beta_1^t}, \quad \hat{\mathbf{v}}_t = \frac{\mathbf{v}_t}{1 - \beta_2^t}$

最终更新:

$\boldsymbol{\theta}_{t+1} = \boldsymbol{\theta}_t - \frac{\alpha}{\sqrt{\hat{\mathbf{v}}_t} + \epsilon} \hat{\mathbf{m}}_t$

典型超参:$\beta_1 = 0.9, \beta_2 = 0.999, \epsilon = 10^{-8}, \alpha = 10^{-3}$。这些数字几乎所有 PyTorch/TensorFlow 代码默认都是它们。

AdamW

Loshchilov 2017 指出:Adam 里的"weight decay"(权重衰减正则化)和学习率耦合不正确。AdamW 解耦,现在是 LLM 训练的默认选择。主书 5.7.4 里 LLaMA 训练配方的 "AdamW" 就是它。

影响与争议

Adam 是深度学习最广使用的优化器——像空气一样默认。但也有争议:

  • - 收敛性:Reddi 2018 "On the Convergence of Adam"证明原版 Adam 在某些简单凸问题上不收敛。提出 AMSGrad 修复(但实用中很少用)
  • - 泛化:在 CV 任务上 SGD+Momentum 常常泛化更好。但在 Transformer 训练上 Adam/AdamW 无替代
  • - 内存:Adam 需要存 $\mathbf{m}, \mathbf{v}$ 各一份——是参数的 2 倍。大模型时代这是显存杀手。ZeRO(主书 5.7.2)就是为了分片这个状态而生
为什么必读:每个 LLM 训练背后都在用 AdamW。这是"默认值"的力量——但知道默认值背后的数学,你才能在遇到训练不稳定时对症下药。

P05XGBoost:表格数据的事实标准

对应主书:第 3 章 4.2.2 树模型

XGBoost: A Scalable Tree Boosting System
作者Tianqi Chen, Carlos Guestrin
机构University of Washington
发表KDD 2016
被引30,000+
链接arXiv:1603.02754

问题:GBDT 的两个瓶颈

传统 GBDT(Friedman 2001)每棵树拟合上一步的残差(对平方损失等价于负梯度)。两个问题:

  1. 只用一阶梯度:优化方向信息不够,收敛慢
  2. 工程层面慢:大数据集下单机训练几小时起,分布式实现缺失

核心方法:二阶泰勒 + 正则化 + 并行化

陈天奇把目标函数在当前模型处做二阶泰勒展开:

$\mathcal{L}^{(t)} \approx \sum_i \left[ g_i f_t(\mathbf{x}_i) + \frac{1}{2} h_i f_t(\mathbf{x}_i)^2 \right] + \Omega(f_t)$

其中 $g_i = \partial L / \partial F$(一阶),$h_i = \partial^2 L / \partial F^2$(二阶),$\Omega(f) = \gamma T + \frac{1}{2} \lambda \sum_j w_j^2$ 是正则(T 是叶子数,$w_j$ 是叶子权重)。

对一棵确定结构的树,闭式解出每个叶子的最优权重:

$w_j^* = -\frac{\sum_{i \in I_j} g_i}{\sum_{i \in I_j} h_i + \lambda}$

分裂增益的闭式:

$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$

有了这个公式,贪心选最大 Gain 的分裂,直到 Gain < 0(或其他停止条件)。详细推导见主书 4.2.2。

工程创新

  • 近似分裂算法:大数据时不枚举所有分裂点,按分位数离散化(LightGBM 后来把这推到极致——直方图算法)
  • 稀疏感知:缺失值自动被分到"默认方向",无需手工填充
  • 列块存储:按特征列压缩存储,排序一次复用多轮
  • 并行:特征级并行(不同特征各自找分裂点)
  • Cache-aware 访问:预取到 CPU cache 提升命中率

Python 使用

import xgboost as xgb
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split

X, y = load_breast_cancer(return_X_y=True)
Xt, Xv, yt, yv = train_test_split(X, y, test_size=0.2)

model = xgb.XGBClassifier(
    n_estimators=500, max_depth=6, learning_rate=0.05,
    reg_lambda=1.0, reg_alpha=0.1,
    early_stopping_rounds=50, eval_metric='logloss',
)
model.fit(Xt, yt, eval_set=[(Xv, yv)])
print("Accuracy:", model.score(Xv, yv))

影响

2015-2020 Kaggle 表格竞赛冠军解决方案 90%+ 含 XGBoost/LightGBM/CatBoost 之一。这三个工具本质都是 XGBoost 思想的变种:

  • LightGBM(微软 2017):直方图加速、leaf-wise 生长替代 level-wise
  • CatBoost(Yandex 2018):原生类别特征处理、有序提升

在工业界,XGBoost 至今仍是表格数据的首选。原因:训练推理都快(百万样本分钟级)、不用 GPU、不需要特征归一化、特征重要性天然可解释。深度学习对表格数据长期没有优势——TabNet、TabTransformer 都没成为主流。

为什么必读:LLM 时代这篇经典仍然重要——因为金融风控、医疗、推荐、网络流量分类等领域,XGBoost 在延迟和可解释性上全面胜过深度学习。知道何时该用它、何时不该用它,是 AI 工程师的基本判断力。

P06Attention Is All You Need:Transformer 的诞生

对应主书:第 4 章 5.2 全节

Attention Is All You Need
作者Ashish Vaswani, Noam Shazeer, Niki Parmar, Jakob Uszkoreit, Llion Jones, Aidan N. Gomez, Łukasz Kaiser, Illia Polosukhin
机构Google Brain, Google Research, University of Toronto
发表NeurIPS 2017
被引140,000+(CS 历史第二)
链接arXiv:1706.03762

问题:RNN 的根本限制

2017 年机器翻译主流是 seq2seq + attention(Bahdanau 2014, Luong 2015)——encoder 用 LSTM 把源句编码成向量序列,decoder 用 LSTM 自回归生成,通过 attention 查看 encoder 输出。效果好,但 LSTM 的时间步串行性让训练无法并行——GPU 的优势发挥不出来,模型规模受限。

Vaswani 等人的问题:能不能完全去掉循环,只用 attention?

核心方法:Scaled Dot-Product Attention

把序列每个位置表示成 Query、Key、Value 三个向量,通过 Q 和所有 K 的相似度加权 V:

$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$

关键细节 $\sqrt{d_k}$ \lt /strong> 缩放:$d$ 维向量点积方差是 $d$,除以 $\sqrt{d}$ 让方差归一,防止 softmax 过饱和(主书 5.2.2 有详细推导)。

Multi-Head Attention

把 Q、K、V 投到 $H$ 个子空间,每个子空间独立做 attention,结果拼接后再投影:

$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H) W^O$
$\text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V)$

多头让模型同时学多种"注意模式"——某些头关注语法邻近,某些关注指代关系,某些关注全局主题。

完整 Transformer Block

flowchart TB X[Input x] --> N1[LayerNorm] N1 --> MHA[Multi-Head
Attention] MHA --> A1((+)) X --> A1 A1 --> N2[LayerNorm] N2 --> FFN[FFN
d_model to 4d to d_model] FFN --> A2((+)) A1 --> A2 A2 --> OUT[Output]

(图中是 Pre-Norm 版本,原论文用 Post-Norm,现代实现大多切换到 Pre-Norm;详见主书 5.2.4。)

位置编码

Attention 本身是"集合运算"——打乱输入顺序输出不变。要给模型位置感,加位置编码:

$\text{PE}(\text{pos}, 2i) = \sin(\text{pos}/10000^{2i/d})$
$\text{PE}(\text{pos}, 2i+1) = \cos(\text{pos}/10000^{2i/d})$

不同频率正弦余弦编码位置信息。后来被 RoPE 替代(见 P09)。

完整 PyTorch 实现

import torch
import torch.nn as nn
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.H = n_heads
        self.d_head = d_model // n_heads
        self.Wq = nn.Linear(d_model, d_model, bias=False)
        self.Wk = nn.Linear(d_model, d_model, bias=False)
        self.Wv = nn.Linear(d_model, d_model, bias=False)
        self.Wo = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, mask=None):
        B, L, D = x.shape
        Q = self.Wq(x).view(B, L, self.H, self.d_head).transpose(1, 2)
        K = self.Wk(x).view(B, L, self.H, self.d_head).transpose(1, 2)
        V = self.Wv(x).view(B, L, self.H, self.d_head).transpose(1, 2)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_head)  # [B, H, L, L]
        if mask is not None:
            scores = scores.masked_fill(mask, float('-inf'))
        attn = torch.softmax(scores, dim=-1)
        out = attn @ V                              # [B, H, L, d_head]
        out = out.transpose(1, 2).reshape(B, L, D)
        return self.Wo(out)


class FFN(nn.Module):
    def __init__(self, d_model, d_ff=None):
        super().__init__()
        d_ff = d_ff or 4 * d_model
        self.fc1 = nn.Linear(d_model, d_ff)
        self.fc2 = nn.Linear(d_ff, d_model)

    def forward(self, x):
        return self.fc2(torch.relu(self.fc1(x)))


class TransformerBlock(nn.Module):
    """Pre-Norm 版本 - 现代 LLM 标准"""
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = MultiHeadAttention(d_model, n_heads)
        self.ln2 = nn.LayerNorm(d_model)
        self.ffn = FFN(d_model)

    def forward(self, x, mask=None):
        x = x + self.attn(self.ln1(x), mask)
        x = x + self.ffn(self.ln2(x))
        return x


class Transformer(nn.Module):
    def __init__(self, vocab_size, d_model=512, n_heads=8, n_layers=6, max_len=5000):
        super().__init__()
        self.tok_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = self._build_pos_enc(max_len, d_model)
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(d_model)
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)

    def _build_pos_enc(self, max_len, d):
        pe = torch.zeros(max_len, d)
        pos = torch.arange(max_len).unsqueeze(1).float()
        div = torch.exp(torch.arange(0, d, 2).float() * -(math.log(10000.0)/d))
        pe[:, 0::2] = torch.sin(pos * div)
        pe[:, 1::2] = torch.cos(pos * div)
        return nn.Parameter(pe.unsqueeze(0), requires_grad=False)

    def forward(self, idx):
        B, L = idx.shape
        x = self.tok_emb(idx) + self.pos_emb[:, :L]
        # Causal mask: 上三角屏蔽
        mask = torch.triu(torch.ones(L, L, dtype=torch.bool, device=idx.device),
                          diagonal=1)
        for block in self.blocks:
            x = block(x, mask)
        x = self.ln_f(x)
        return self.lm_head(x)                      # [B, L, vocab]

# 使用
model = Transformer(vocab_size=10000)
x = torch.randint(0, 10000, (2, 50))   # batch 2, seq 50
logits = model(x)                       # [2, 50, 10000]

这大约 70 行就实现了 Transformer 的核心。真正的生产实现(LLaMA、Qwen)在此基础上有十几个工程优化:RoPE 替代正弦位置编码、RMSNorm 替代 LayerNorm、SwiGLU 替代 ReLU FFN、GQA 替代 MHA 等,但骨架 7 年没变。

实验结果

原论文在 WMT 2014 英-德翻译上:BLEU 28.4,比 Google NMT(基于 LSTM)27.3 提升 1.1 点,且训练时间只要 1/10。这是关键——不是效果好一点,是快到能训更大模型。

影响

Attention Is All You Need 是 AI 历史最重要的几篇论文之一。7 年后(2024)CS 领域只有 ResNet 的引用数超过它。

但它的直接贡献只是机器翻译 1.1 BLEU 的提升——真正改变世界的是它解锁的可能性:

  1. BERT(2018)和 GPT(2018):用 Transformer 做预训练
  2. GPT-3(2020):证明 Transformer 能 scale 到百亿参数并产生涌现能力
  3. ViT(2020):Transformer 搬到 CV
  4. ChatGPT(2022):Transformer + RLHF 变产品
  5. 一切现代 LLM(2023+):LLaMA、Claude、Qwen、DeepSeek 全是 Transformer 变种

一个工程细节的有趣后话

原论文 3.2.2 有一句话:"We found that the Transformer with sinusoidal position encodings performs similarly to learned position embeddings"。后来证明这句话过于乐观——学习式位置编码在超出训练长度时完全失效,正弦位置编码外推也差。这个细节被 RoPE(P09)真正解决,5 年之后。

这也是论文阅读的价值:原作者也会错。了解这些 subtlety 让你对领域演化有更清晰的认识。

为什么是 3 页不是 1 页:如果只读一篇论文来理解现代 AI,就是这一篇。搞懂 attention 机制是所有后续工作的入场券。建议配合 The Annotated Transformer(Harvard NLP)和 Andrej Karpathy 的"Let's build GPT"视频一起吃透。

P07BERT 与 GPT:预训练时代的双子星

对应主书:第 4 章 5.2.5(Encoder/Decoder 分野)· 4.5.2(自监督预训练)

BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding
作者Jacob Devlin, Ming-Wei Chang, Kenton Lee, Kristina Toutanova(Google)
发表NAACL 2019 · arXiv:1810.04805 · 被引 110,000+
Language Models are Few-Shot Learners(GPT-3)
作者Tom Brown 等(OpenAI)
发表NeurIPS 2020 · arXiv:2005.14165 · 被引 35,000+

BERT 和 GPT-3 分属不同技术路线,但一起定义了"预训练时代"。放在一起比较最清晰。

BERT:双向 Encoder + MLM

核心创新:掩码语言模型(Masked Language Modeling)。

传统语言模型是单向的(从左到右预测下一个词)。BERT 随机遮住 15% 的 token,让模型基于双向上下文预测被遮的内容。每个位置的表示都能"看到"整个句子。

flowchart LR I["The [MASK] sat on the mat"] --> ENC[Transformer Encoder
双向 self-attention] ENC --> P["[MASK] = cat (prediction)"]

第二个训练目标 NSP(Next Sentence Prediction):预测两句话是否连续。后续工作(RoBERTa 2019)证明 NSP 其实没用,但 MLM 留下来了。

BERT 的范式

BERT 开启"预训练 + 微调"模式:

  1. 在海量无标注文本(Wikipedia + BookCorpus,~3.3B tokens)做 MLM 预训练
  2. 下游任务微调:分类、问答、NER 等,只要加一层输出头

这个范式把各种 NLP 任务的 SOTA 一次性刷新。BERT-Base(110M 参数)、BERT-Large(340M)——当时看是"大模型",和今天比是小模型。

GPT-3:单向 Decoder + 只看 few-shot

GPT-3 走的是相反路线:

  • - 只用 Decoder(因果语言模型,从左到右预测下一个 token)
  • - 不微调——把任务说明和几个例子写在 prompt 里,模型在"上下文中学习"
  • - 规模暴涨:175B 参数,是 BERT-Large 的 500 倍

GPT-3 的关键发现:In-Context Learning

GPT-3 论文的真正贡献不是架构创新——架构和 GPT-2 几乎一样。贡献是证明了:模型 scale 到一定程度,会涌现出"从几个例子学新任务"的能力。

例子:

Translate English to French:
sea otter => loutre de mer
plush giraffe => girafe peluche
cheese => ???

GPT-3 在看过 2 个例子后,正确输出 "fromage"——而且能推广到数学、推理、代码等任务。这是 LLM 从"语言模型"变成"通用推理器"的起点。

为什么 Decoder-only 最终赢了

主书 5.2.5 分析过,主要因素:

  1. Decoder-only 训练更简单——单一目标,所有文本都是训练数据
  2. In-context learning 让"一个模型做所有任务"成为现实
  3. Scaling 性质对 decoder-only 更友好——GPT-3 以后这是共识
  4. 生成能力天然包含理解能力(能写好回答,就得先看懂问题)

BERT 系的 encoder 模型今天主要用作"文本编码器"——embedding 模型(BGE、E5)、reranker 等。RAG 的检索阶段大多还是 encoder 模型。

影响

BERT 开启了预训练范式——今天所有 LLM 都继承这个思想。GPT-3 证明了scaling 的威力——直接导向 ChatGPT、GPT-4。这两篇是 LLM 历史的双子星。

为什么必读:BERT 让你理解"预训练-微调"范式。GPT-3 让你理解"涌现能力"和"in-context learning"——这两个概念是后续整个 LLM 时代的核心假设。

P08Scaling Law 与 Chinchilla:训练大模型的经济学

对应主书:第 4 章 5.1.1 · 4.5.2

Scaling Laws for Neural Language Models
作者Jared Kaplan 等(OpenAI)
发表arXiv:2001.08361(2020)
Training Compute-Optimal Large Language Models(Chinchilla)
作者Jordan Hoffmann 等(DeepMind)
发表NeurIPS 2022 · arXiv:2203.15556

问题:给定算力预算,怎么分配到参数和数据?

训练一个大模型,总算力 $C \approx 6 \cdot N \cdot D$($N$ 参数数,$D$ 训练 token 数)。假设你有固定的 $C$(比如 1e25 FLOPs),是该训大模型小数据,还是小模型大数据?

Kaplan(2020):Loss 是幂律

OpenAI 跑了上百个不同大小的模型,发现 loss 随参数和数据都是幂律下降:

$L(N, D) \approx E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$

$E$ 是不可约熵,$\alpha \approx 0.07, \beta \approx 0.28$。

Kaplan 的关键结论(后来被证明错误):在固定算力下,最优分配是参数 $N$ 增长远快于数据 $D$。这启发了 GPT-3 175B 的设计——只训 300B tokens(按 Kaplan 预测已经接近最优)。

Chinchilla(2022):Kaplan 错了

DeepMind 重做实验,用更广的 $N, D$ 组合。新拟合公式:

$L(N, D) = E + \frac{A}{N^{0.34}} + \frac{B}{D^{0.28}}$

关键:$\alpha$ 从 0.07 变成 0.34——这意味着参数和数据应该大致等比例增长。

具体结论:最优点 tokens-to-parameters 比约 20:1。

模型参数训练 tokens实际比例是否 Chinchilla 最优
GPT-3175B300B1.7:1❌ 数据严重不足
Chinchilla70B1.4T20:1✅ 最优
LLaMA-1 65B65B1.4T21.5:1✅ 接近最优
LLaMA-3 70B70B15T214:1推理效率过度,超 Chinchilla 多训

Chinchilla 的实验证明

Chinchilla(70B 参数、1.4T tokens 训练)和 Gopher(280B 参数、300B tokens)——同等算力下,Chinchilla 在 MMLU 等基准上全面超越 4 倍大的 Gopher。"小而精"击败"大而少",是 Scaling Law 研究的重要修正。

训练最优 vs 推理最优

Chinchilla 算的是"训练时 loss 最低"。但实际部署时,模型要推理千万次,推理效率也是优化目标。

LLaMA-2/3 系列意识到这点:刻意训练超过Chinchilla 最优——LLaMA-3-70B 训 15T tokens(214:1),虽然训练超"最优",但得到一个"参数少、能力强"的模型,推理更便宜。这叫"over-training for inference"。

影响

这两篇论文从根本上改变了 LLM 训练决策:

  1. GPT-3 之后"只堆参数"的路线被纠正
  2. 数据收集和清洗成为和模型架构同等重要的问题
  3. LLaMA 系列的数据量(1.4T → 2T → 15T)直接源于这里的洞察
为什么必读:如果你要理解 2022 年后每次大模型发布时"参数 vs 数据"的讨论,必须读这两篇。也是理解"为什么现在数据这么贵"的起点——Chinchilla 的结论让所有公司都在抢数据。

P09RoPE:旋转位置编码

对应主书:第 4 章 5.3.2 RoPE 深挖

RoFormer: Enhanced Transformer with Rotary Position Embedding
作者Jianlin Su, Yu Lu, Shengfeng Pan, Ahmed Murtadha, Bo Wen, Yunfeng Liu
机构追一科技
发表Neurocomputing 2024 · arXiv:2104.09864(2021)
被引4,500+(快速增长中)

问题:绝对位置编码不能外推

原始 Transformer 用正弦绝对位置编码,GPT-2 用学习式绝对位置编码。两者都有同一问题——训练时见过 0-2047 位置,推理时见 2048+ 就失效。学习式更糟(超范围直接爆),正弦好一点但也不 work。相对位置编码(T5 bias、ALiBi)有所改善,但表达力有限。

核心思想:旋转而非相加

RoPE 不给输入加位置向量,而是对 Q 和 K 做旋转。核心洞察:两个向量先分别旋转再做点积,结果只依赖于它们之间的相对角度。

$\mathbf{q}_m^\top \mathbf{k}_n = (R_m \mathbf{q})^\top (R_n \mathbf{k}) = \mathbf{q}^\top R_m^\top R_n \mathbf{k} = \mathbf{q}^\top R_{n-m} \mathbf{k}$

(因为旋转矩阵 $R$ 有 $R_m^\top R_n = R_{n-m}$)

所以 attention 分数自然只依赖相对位置 $n - m$。这正是相对位置编码想要的性质,且实现更优雅。

2D 旋转矩阵

对每对相邻维度做 2D 旋转:

$R_m^{(i)} = \begin{pmatrix} \cos m\theta_i & -\sin m\theta_i \\ \sin m\theta_i & \cos m\theta_i \end{pmatrix}, \quad \theta_i = 10000^{-2(i-1)/d}$

不同维度用不同频率 $\theta_i$:低维高频、高维低频——每个维度捕捉不同尺度的位置差异。

复数视角(更优雅)

把 2D 向量看作复数 $q = q_1 + iq_2$。旋转 $m\theta$ 等于乘 $e^{im\theta}$:

$q_m = q \cdot e^{im\theta}, \quad k_n = k \cdot e^{in\theta}$
$\langle q_m, k_n \rangle = \text{Re}(q \cdot e^{im\theta} \cdot \overline{k \cdot e^{in\theta}}) = \text{Re}(q \bar{k} \cdot e^{i(m-n)\theta})$

只依赖 $m - n$。这是 RoPE 数学上最优雅的表达。

实现

def build_rope_cache(d_head, max_len, base=10000, device='cuda'):
    theta = 1.0 / (base ** (torch.arange(0, d_head, 2).float() / d_head))
    t = torch.arange(max_len, device=device).float()
    freqs = torch.outer(t, theta.to(device))            # [max_len, d_head/2]
    return torch.polar(torch.ones_like(freqs), freqs)   # 复数形式

def apply_rope(x, freqs_cis):
    # x: [B, H, L, d_head]
    x_complex = torch.view_as_complex(
        x.float().reshape(*x.shape[:-1], -1, 2)
    )
    x_rotated = x_complex * freqs_cis[:x.shape[-2]]
    return torch.view_as_real(x_rotated).flatten(-2).type_as(x)

# 用法: 在 attention 里应用到 Q, K (不作用 V)
Q = apply_rope(Q, freqs_cis)
K = apply_rope(K, freqs_cis)
# V 不做 RoPE — 它携带内容, 不应带位置

RoPE 的衍生技术

RoPE 的连续三角函数性质让它有"外推潜力"——下面几项技术把 RoPE 训练的模型扩展到远超训练长度:

  • PI(Position Interpolation, Meta 2023):压缩位置到训练范围内,2-4× 外推
  • NTK-aware:只压缩高频维度,保留低频——无需微调就能外推
  • YaRN(2023):分频段精细化 + 注意力温度调节,1B token 微调可 32× 外推
  • LongRoPE(2024):每维度独立搜索最优缩放,2M token 可行

影响

RoPE 今天是开源 LLM 的事实标准:LLaMA、Qwen、DeepSeek、Mistral、ChatGLM 全部用 RoPE。原因:

  1. 相对位置的数学性质天然
  2. 零额外参数
  3. 外推友好(配合 YaRN 等)
  4. 实现简单(< 30 行代码)

有意思的是,RoPE 作者苏剑林(追一科技)不是大公司研究员,这篇论文最初在他个人博客发表。这是开源社区影响力的一个缩影——好的想法不一定来自大公司。

为什么必读:长上下文(128K、1M)是 2024-2026 LLM 的主战场之一,RoPE + YaRN 是这条路的底层技术。读懂 RoPE 才能理解为什么今天的模型能支持这么长的上下文。
